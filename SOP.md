# 本地阿拉伯语字幕提取 SOP

全本地、免费、离线。基于 Whisper large-v3 + faster-whisper。

## 一、日常用法（3步）

1. 把视频/音频文件丢进 `inbox/` 文件夹。
2. 打开终端，在项目目录下运行：
   ```bash
   python3 run.py
   ```
   （会处理 inbox/ 里所有文件。也可只处理一个：`python3 run.py /path/to/video.mp4`）
3. 去 `outputs/` 拿结果，每个文件生成三样：
   - `<名字>.srt` —— 字幕，毫秒级时间戳
   - `<名字>.txt` —— 纯文本
   - `<名字>.warnings.txt` —— **需人工复查的段落清单**（务必看）

## 二、拿到结果后必做的一步：看 warnings

流水线不会替你判断难听清的内容，它只会把**可疑段落标出来**，绝不静默删除（避免误伤真内容）。warnings 里会列三类：
- **疑似幻觉/套话**：如 "订阅频道 اشتركوا في القناة" 这类模型在音乐/静音段编出来的话。
- **超长段（>10秒）**：往往是幻觉或漏切，重点核对。
- **重复段**：模型卡住重复输出。

对着这些时间点回放原视频，人工确认或修正。

## 三、这套流水线做了什么（为什么比裸跑 Whisper 好）

| 环节 | 作用 |
|---|---|
| 音频规范化(16kHz 单声道) | 统一输入，减少解码异常 |
| 词级时间戳 | 拿到毫秒级时间轴，而非整秒 |
| 反幻觉参数 | 关闭上文依赖(防错误滚雪球)、温度回退、低置信度/压缩比阈值 |
| VAD 静音过滤 | 跳过静音段，减少空白处瞎编 |
| 后处理告警 | 自动标出幻觉/超长/重复段供复查 |

## 四、背景音乐重的视频：开人声分离

**最大短板是背景音乐/音效重的视频**：Whisper 容易在音乐段幻觉（那个30秒"订阅频道"就是）。
本流水线内置了**可选的人声分离**（Demucs）——先把背景音乐剥掉、只留人声，再转录。

```bash
# 一次性安装分离依赖(会装 torch，几百MB)
pip install -r requirements-separation.txt
# 之后按需在跑的时候开启：
SEPARATE_VOCALS=1 python3 run.py
```

- 首次运行自动下载分离模型（约80MB）。
- 需要 `soundfile` 让 Demucs 能读音频（不必装系统 ffmpeg）。
- 分离万一失败会自动**回退到原音频**，不会中断。
- 实测：这一步干掉了那段30秒"订阅频道"幻觉、恢复了真实对话。
- **只对"音乐/噪音"类错误有效**；本身听不清的方言口语仍需人工核对。
- 用法建议：音乐/音效重的视频才开；纯说话的视频不用开（还更快）。

## 五、方言注意

素材多为海湾/沙特口语(عامية)。字幕做**忠实口语记录**时，`لازم`(必须)、`يروحون`(去)、`قبل ينتهي`(省略أن)等口语说法都是对的，不要按标准语(فصحى)去"纠错"。只需修真正的错：拼错、乱码词、人名错、漏字。

## 六、模型与复用

- 默认用 `large-v3`，首次运行自动下载（约3GB），之后离线复用。
- 若已在别处下过模型，用环境变量复用、免重复下载：
  ```bash
  export WHISPER_MODEL=/path/to/faster-whisper-large-v3
  ```
- 语言默认 `ar`，可改：`export WHISPER_LANG=xx`
